# Chapter 1:Reliable, Scalable, and Maintainable Applications

### Basic Definitions
* **Reliability**: able to work in spite of hardware faults, software faults, and human error or malicious use
* **Scalability**: able to handle increasing volumes (data, traffic, complexity, etc)
* **Maintainability**: enables people to adapt the system to new requirements 
* **fault**: a component of system deviates from spec
* **failure**: system as a whole stops providing required service to user

### Limiting Factors of Modern Systems
* data-intensive apps

## Reliability

### Hardware Faults
* Signs of Hardware Faults:
    * 500s errors when sending API calls
    * isolated incidents
* Typical Examples:
    * power grid black-outs
    * hard disk crash
    * RAM losing efficacy
* Typical solutions:
    * adding redundancy: backup generators and other backups
    * software fault-tolerance techniques to handle loss of entire machines
        * e.g exponential backoff to handle 500 internal server errors
        * rolling upgrades to deploy on one machine at a time in a system instead of having downtime on the entire system

### Software Errors
* Signs of Software Errors
    * Systemic errors that are happening across multiple instances or components in the system
    * tends to repeat
    * triggered by specific input
    * often receiving 400s or 500s errors when sending API calls
* Examples:
    * software bug that causes fault or failure when application receives specific input
    * a runaway process using up a shared resource: CPU, memory, disk space, network bandwidth
    * a service slows down, becomes unresponsive, or starts returning corrupted responses
    * cascading failures
* Solutions:
    * thorough testing
    * constant measuring, monitoring, and analyzing systems in production
